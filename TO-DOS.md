- **Field highlighting** - schema-required inputs now show a soft red border + pale-red background when empty, clearing as soon as the user types. Implemented via native HTML `required` attribute on every input mapped to a schema-required property (adapter model + ip/com, port id, iTach2CC port position, method id/command, param id/value, scene id/name, response filter name/regex/trigger), with a single `.builder-panel input:required:invalid` rule in `BuilderPanel.vue`. - **Friendly error messages: path translation** - Validation warnings used to read like `/adapters/0/ports/0/id does not match the required format`; now they read like `GenericNetworkAdapter 192.168.1.10 / port 'cc_light' / id does not match the required format`. Added `friendlyPath()` in `src/validation/validateProfile.js` which walks the JSON pointer against the parsed profile, replacing numeric indices with `<model> <address>` for adapters, `port '<id>'` / `method '<id>'` / etc. for nested arrays, and `Nth port` / `Nth method` ordinals when the item has no id yet. Applied to schema errors, cross-ref errors, duplicate-id errors, and quirk errors — both the top-level `path` field and any embedded path references inside `message`. Schema "required" / "additionalProperties" messages reworded to lead with the path (`X is missing required field 'id'`) instead of trailing it. Line-number surfacing is split out to a follow-up.

- **Builder error cascade** - Every container above an empty required input (adapter / port / method / param card, scene / rule / filter card, free-form style row) now picks up a dashed-red outline via CSS `:has(input:required:invalid)` so the user can trace from the section header down to the leaf input. Uses `outline` (not `border`) with `outline-offset: -1px` so containers without a baseline border (port-card, style-row) don't reflow when the indicator toggles. Pure CSS — no JS state to keep in sync; flips on/off on every keystroke automatically.

- **Activity log** - Log lives in a right-side drawer overlay inside the preview pane, toggled via a `receipt_long` button in the preview toolbar next to theme / fullscreen. Closed by default; button shows an entry-count badge when there's activity. Drawer is `position: absolute` against `#preview`; the scroll moved from `#preview` itself onto a new `.preview-content` wrapper so the drawer stays pinned while controls scroll underneath. Each scene / event / control click logs a summary (`Scene "Power On" → 3 commands`) with one detail line per resolved command (`projector.power.on → 192.168.1.10: power=on\r`); unresolved refs are surfaced inline (`unresolved: no matching port`) and bump the entry to warn level. The Output card was also extended to show the ref tag (`projector.power.on →`) before each command so users see the mapping at a glance, not just the final bytes. JSON edits log a single debounced "JSON modified" entry (600 ms), validation warnings log when the count goes up, and parse errors log immediately. Capped at 200 entries in-memory (newest first); Clear button in the drawer header. Implemented in `transformProfile.js` (resolveCommandRefs now returns `{ ref, address, command }` or `{ ref, error }` per ref) and `HomeView.vue` (logEntries data, pushLog/clearLog/logTrigger methods, watchers on `json` / `validationWarnings` / `parseError`).
